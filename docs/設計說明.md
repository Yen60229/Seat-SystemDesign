# 員工座位系統 設計文件

## 目標

實作人資部門使用的員工座位安排系統：

1. 顯示各樓層座位（從 DB 讀取，顏色區分空位/已佔用，已佔用顯示員編）
2. 調整座位（每人限一位、下拉選員工、點空位變「選擇中」顏色、可清除佔用、送出寫回 DB）
3. 三層式架構（Web Server + Application Server + 關聯式資料庫），後端分展示/業務/資料/共用層
4. 技術：Vue.js、Spring Boot、RESTful API、Maven、Stored Procedure、Transaction、DDL/DML 放 `/DB`、防 SQL Injection 與 XSS

## 技術選型

| 項目 | 選擇 | 理由 |
|---|---|---|
| 資料庫 | MySQL 8（Docker 容器） | 免費、SP 支援完整、面試官易重現 |
| 後端 | Spring Boot 3.4 + JDK 17 + Maven | 題目要求；JDK 17 為 Boot 3 最低需求 |
| DB 存取 | Spring JDBC `SimpleJdbcCall` | 直接呼叫 SP、參數化防 SQL Injection |
| 前端 | Vue 3 + Vite + axios | 題目要求 Vue.js |

## 系統架構

```
瀏覽器 (Vue 3 SPA)
   │ HTTP/JSON (RESTful)
Spring Boot (內嵌 Tomcat = Web Server + Application Server)
   ├─ controller  展示層：REST API、輸入驗證
   ├─ service     業務層：座位指派規則、@Transactional
   ├─ dao         資料層：SimpleJdbcCall 呼叫 Stored Procedure
   └─ common      共用層：DTO、統一回應格式、全域例外處理
   │ JDBC (只透過 SP)
MySQL 8 (Docker)
```

正式繳交時前端 `npm run build` 產出靜態檔放入 Spring Boot `static/`，單一服務即可展示；開發時 Vite dev server 以 proxy 轉發 `/api`。

## 資料庫設計（/DB 資料夾提供 DDL + DML）

### 資料表

**EMPLOYEE 員工資料**
| 欄位 | 型別 | 說明 |
|---|---|---|
| EMP_ID | CHAR(5) PK | 員編，固定 5 碼 |
| NAME | VARCHAR(50) | 姓名 |
| EMAIL | VARCHAR(100) | 電子郵件 |
| FLOOR_SEAT_SEQ | INT NULL, FK→SEATING_CHART, UNIQUE | 座位序號；NULL=未配位；UNIQUE 保證一座位僅一人 |

**SEATING_CHART 樓層座位表**
| 欄位 | 型別 | 說明 |
|---|---|---|
| FLOOR_SEAT_SEQ | INT PK AUTO_INCREMENT | 序號 |
| FLOOR_NO | INT | 樓層編號 |
| SEAT_NO | INT | 座位編號 |
| (UNIQUE) | (FLOOR_NO, SEAT_NO) | 同樓層座位不重複 |

**SEAT_CHANGE_LOG 座位異動紀錄**（第二張異動表，與 EMPLOYEE 同交易寫入，滿足「同時異動多個資料表需 Transaction」）
| 欄位 | 型別 | 說明 |
|---|---|---|
| LOG_SEQ | BIGINT PK AUTO_INCREMENT | 流水號 |
| EMP_ID | CHAR(5) | 員編 |
| OLD_FLOOR_SEAT_SEQ / NEW_FLOOR_SEAT_SEQ | INT NULL | 異動前後座位 |
| CHANGE_TIME | DATETIME | 異動時間 |

### Stored Procedures

| SP | 用途 |
|---|---|
| `SP_GET_FLOORS()` | 取得所有樓層清單 |
| `SP_GET_SEATS_BY_FLOOR(p_floor_no)` | 取得該樓層全部座位 + 佔用者員編/姓名（LEFT JOIN EMPLOYEE） |
| `SP_GET_ALL_EMPLOYEES()` | 員工下拉選單資料（含目前座位） |
| `SP_ASSIGN_SEAT(p_emp_id, p_floor_seat_seq)` | 指派座位：檢查座位未被佔用 → 更新 EMPLOYEE.FLOOR_SEAT_SEQ + 寫 SEAT_CHANGE_LOG；衝突時 SIGNAL 錯誤 |
| `SP_CLEAR_SEAT(p_floor_seat_seq)` | 清除座位：將佔用者 FLOOR_SEAT_SEQ 設 NULL + 寫 LOG |

`/DB/01_DDL.sql`：建庫、建表、建 SP。`/DB/02_DML.sql`：種子資料（3 個樓層、每層 ~20 座位、~15 名員工部分已配位）。

## REST API

| Method | Path | 說明 |
|---|---|---|
| GET | `/api/floors` | 樓層清單 |
| GET | `/api/floors/{floorNo}/seats` | 該樓層座位（含佔用狀態） |
| GET | `/api/employees` | 員工清單（下拉選單用） |
| POST | `/api/seats/batch` | 批次送出座位異動 `[{empId, floorSeatSeq, action: ASSIGN/CLEAR}]`，Service 層 `@Transactional` 包整批：任一筆失敗就整批還原 |

統一回應格式 `ApiResponse<T> { code, message, data }`；錯誤由 `@RestControllerAdvice` 全域處理。

## 業務規則

- 每位員工只能佔用一個座位：DB 層 UNIQUE 約束 + SP 內檢查 + 前端送出前檢查
- 員編固定 5 碼：後端 Bean Validation `@Pattern("^[A-Z0-9]{5}$")` + 前端僅能從下拉選單選擇
- 送出採批次：前端暫存多筆異動，按「送出」一次 POST，後端單一交易寫入

## 前端（Vue 3）

單頁畫面：
- 樓層 tab 切換
- 座位格狀圖：綠=空位、紅=已佔用（顯示員編）、黃=選擇中（pending 異動）
- 員工下拉選單（選員工 → 點空位 = 預定指派；點已佔用座位 → 可標記清除）
- 「送出」按鈕：POST `/api/seats/batch` 後重新載入
- XSS 防護：Vue 模板插值預設轉義，不使用 `v-html`

## 安全

- SQL Injection：所有 DB 存取走 SP + `SimpleJdbcCall` 參數化，無字串拼接 SQL
- XSS：前端不用 `v-html`；後端驗證輸入格式（員編 5 碼、floorSeatSeq 為整數）
